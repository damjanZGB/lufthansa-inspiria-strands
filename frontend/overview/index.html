Project Overview

Daisy is an AWS Bedrock Agent solution that transforms traveler intent into concrete Lufthansa Group flight options. It integrates an Agent (with action groups), a Python Lambda executor, a minimal Node proxy for Bedrock + Amadeus tooling, and lightweight frontends.

Components & Roles

Agent (Bedrock)

Hosts instructions and routes calls to action groups. Ensures tool‚Äëfirst behavior and consistent presentation (THEN lines, sections, flight number format).

Models (LLM)

The foundation model used by the Agent is configured in AWS Bedrock per agent version and alias. This repo does not hard-code a model ó check Bedrock for the authoritative source.

Fetch model config via CLI:

aws bedrock-agent list-agent-aliases --agent-id JDLTXAKYJY --region us-west-2
aws bedrock-agent list-agent-versions --agent-id JDLTXAKYJY --region us-west-2
aws bedrock-agent get-agent --agent-id JDLTXAKYJY --agent-version <97|98|99> --region us-west-2

Record the chosen model per alias in docs (handoff.md). Avoid calling Bedrock directly from the browser/UI.

Lambda: daisy_in_action

Executes function‚Äëdetails and OpenAPI actions: resolves IATA, parses time phrases, calls Amadeus via the proxy, and returns simplified offers plus human‚Äëreadable message text.

Proxy (proxy.mjs)

Minimal gateway for Bedrock and tools. Signs Bedrock Invoke, normalizes event streams, exposes Amadeus/IATA/Time adapters, and implements text fallback. CORS‚Äëgated and transcript‚Äëaware.

Frontends

Four identical UIs (config‚Äëonly variants). Render chat, sanitize formatting, support PDF creation from itineraries, and log transcripts. Now rely solely on text from proxy.

Amadeus

Flight Offers Search v2.8/2.9. Proxy forwards GET query params (origin/destination, dates, passengers, cabin, LH filters, currency, nonstop, max) and adds Accept: application/vnd.amadeus+json.

Observability

CloudWatch captures Lambda/Proxy telemetry. Optional S3 debug captures full tool I/O payloads. Replay scripts summarize failures, formatting, placeholders, and tool‚Äëcall counts.

Formatting Guarantees

  • THEN lines for connections; flight numbers compact (e.g., LH612).
  • Sections: DIRECT FLIGHTS, CONNECTING FLIGHTS.
  • No placeholders: no ‚ÄúEUR X.XX‚Äù, ‚ÄúAirport Name N‚Äù, ‚ÄúNotes: ‚Ä¶‚Äù.

Fallback Logic (Proxy)

  • If no streamed text ‚Üí use functionResponse TEXT.
  • If streamed text is a short/heading line ‚Üí replace with functionResponse TEXT.
  • Frontends do not parse finalResponse; they rely on proxy‚Äôs text.

Troubleshooting

No Text Response

  • Proxy now falls back to functionResponse TEXT on empty/heading-only streams.
  • Confirm proxy logs show usedFunctionResponseFallback: true.

Amadeus Issues

  • Check creds and Accept header.
  • Verify includedAirlineCodes forwarded when needed.
  • Timeouts: tune AMADEUS_TIMEOUT_MS.

Formatting Drift

  • Run replay suite and inspect format_ok / placeholders.
  • Review instruction ASCII variants in aws/.
